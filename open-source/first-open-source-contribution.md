# First Open Source Contribution

**Date:** 2026-03-10

## What I learned

Contributing to open source is easier than it seems. Many repos have "good first issue" labels specifically for newcomers.

## My first PR

- **Repo:** [lingdojo/kana-dojo](https://github.com/lingdojo/kana-dojo)
- **PR:** [#7962](https://github.com/lingdojo/kana-dojo/pull/7962)
- **What I did:** Added a Japanese proverb to the community content
- **Time taken:** < 5 minutes

## Key takeaways

1. Look for repos with `good first issue` labels
2. Start with content/docs changes — no setup needed
3. Follow the repo's contributing guide and commit conventions
4. Fork → Branch → Change → PR is the standard workflow

## Good resources for finding issues

- [goodfirstissue.dev](https://goodfirstissue.dev)
- [up-for-grabs.net](https://up-for-grabs.net)
- GitHub search: `label:"good first issue" language:TypeScript state:open`
